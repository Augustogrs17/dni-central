Guia rápido para o time DNI

Dicas
1
Selecionar a pasta de projetos
Na aba Projetos, clique no ícone de pasta (📁) ao lado da barra de caminho, ou clique diretamente na barra cinza. Navegue até a pasta de rede onde os arquivos .xlsm estão armazenados.

O caminho é salvo automaticamente — na próxima abertura do aplicativo, a pasta já estará configurada.
💡 A pasta pode ser local (C:\Projetos) ou de rede (\\servidor\Projetos). Qualquer caminho acessível pelo seu usuário funciona.
2
Carregar os projetos
Com a pasta selecionada, clique em Carregar. O aplicativo lê todos os arquivos .xlsm da pasta e exibe os projetos na tabela.

Projetos com status CANCELADO são ignorados automaticamente.
3
Alternar tema claro / escuro
Clique no botão ☀ / 🌙 no canto superior direito da barra de título para alternar entre o tema escuro e o tema claro. A preferência é salva automaticamente.
1
Cards de resumo (barra superior da tabela)
Após carregar, a barra de cards mostra a contagem de projetos por status (Preliminar, Conceito, Alpha, Beta, Efetivado, etc.).

Clique em qualquer card para filtrar a tabela por aquele status. Clique em Total para limpar o filtro de status.
2
Filtros
Use a barra de filtros para refinar a visualização:

• Status — filtra por fase do projeto (Preliminar, Conceito, etc.)
• Líder — mostra apenas projetos de um determinado líder técnico
• Categoria — filtra por categoria (Estratégicos, S&OP, etc.)
• Buscar — pesquisa livre por DEP, assunto, modelo ou qualquer campo

Os filtros são cumulativos — você pode combinar todos ao mesmo tempo. Clique em ✕ Limpar para resetar todos os filtros de uma vez.
💡 Para ver todos os projetos de um líder, use o filtro Líder e selecione o login desejado.
3
Ordenação da tabela
Clique no cabeçalho de qualquer coluna para ordenar a tabela por aquele campo. Clique novamente para inverter a ordem (↑ ascendente / ↓ descendente).

Colunas disponíveis para ordenação: DEP, Assunto, Modelo, Líder, Fase, Categoria, Status.
4
Projetos Efetivados
Quando um projeto é efetivado, a coluna Status exibe o badge Efetivado acompanhado da data de efetivação no formato dd/mm/aaaa.

Módulo em desenvolvimento
O módulo de Itens DI permitirá extrair todos os itens com SIT = DI dos projetos ativos, classificados por tipo: COMP (componente comprado), FABR (fabricado) e BENEF (beneficiado).

Funcionalidades previstas:
• Extração de todos os projetos ativos em paralelo
• Filtro por DEP ou por Líder Técnico
• Deduplicação por código dentro do mesmo projeto
• Geração de planilha Excel com abas por tipo
• Envio automático por e-mail às sextas-feiras
🔜 Disponível na versão 0.2.0
—
Módulo em desenvolvimento
O módulo PPAP cruzará os itens DI/COMP com os registros exportados do Monday (board Qualidade), identificando para cada item:

• Se o PPAP está disponível (documento anexado no Monday)
• O fornecedor responsável
• O status da avaliação: Disponível para Análise, Em Análise, Aprovado, Reprovado ou Não Registrado
• O status da embalagem

O export do Monday deve ser salvo localmente antes de iniciar o cruzamento. O caminho do arquivo é selecionado diretamente no aplicativo.
🔜 Disponível na versão 0.3.0
💡
Atalho de teclado
Pressione F1 para abrir este manual a qualquer momento. Pressione Esc para fechar.
💡
Projetos cancelados não aparecem
O aplicativo ignora automaticamente todos os arquivos .xlsm cujo campo Status do Projeto (célula Q17) esteja como CANCELADO. Nenhuma configuração adicional é necessária.
💡
Configurações salvas automaticamente
A pasta de projetos selecionada e a preferência de tema são salvas automaticamente em um arquivo dni_central_config.json ao lado do executável. Ao reabrir o aplicativo, tudo estará configurado como você deixou.
💡
Arquivos temporários do Excel
Arquivos que começam com ~$ (gerados pelo Excel quando um arquivo está aberto) são ignorados automaticamente pelo aplicativo, evitando erros de leitura.
